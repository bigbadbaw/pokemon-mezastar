Review all staged and unstaged changes, then help prepare a pull request.

Steps:
1. Run `git diff` and `git diff --cached` to see all changes
2. Summarize what was changed and why
3. Check for any issues: lint errors, type errors, missing tests
4. Generate a PR title following conventional commits format
5. Generate a PR description with:
   - Summary of changes
   - Screenshots/recordings needed (flag if UI changed)
   - Testing done
   - Any breaking changes
6. Create the commit(s) with good messages
7. Push to the current branch
